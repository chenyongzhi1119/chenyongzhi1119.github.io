Claude Code 是 Anthropic 推出的 AI 编程助手 CLI 工具，能够直接在终端中读写代码、执行命令、操作 Git 等。本文整理了新手上手时最实用的技巧。

<!-- more -->

## 安装与启动

```bash
npm install -g @anthropic-ai/claude-code
claude
```

首次运行会引导你完成 API Key 配置或 Claude.ai 账号授权。

---

## 技巧一：用自然语言描述任务，而不是写命令

Claude Code 的核心优势是理解意图。不需要记住具体命令，直接说你想做什么：

```
# 不需要这样：
git log --oneline --graph --all

# 直接说：
帮我查看最近 10 次提交记录，用图形展示分支
```

对于复杂任务同样适用：

```
帮我找出项目中所有超过 200 行的函数，列出文件路径和行号
```

---

## 技巧二：善用 CLAUDE.md 配置项目上下文

在项目根目录创建 `CLAUDE.md`，Claude Code 每次启动都会自动读取它。可以写入：

- 项目架构说明
- 常用命令（构建、测试、部署）
- 代码规范和注意事项

```markdown
# CLAUDE.md

## 构建命令
- 开发：`npm run dev`
- 部署：`npm run build && npm run deploy`

## 注意事项
- 所有接口必须有错误处理
- 数据库操作使用事务
```

这样就不用每次重复解释项目背景了。

## 技巧三：用 `/` 斜杠命令提高效率

Claude Code 内置了一批斜杠命令，常用的有：

| 命令 | 作用 |
|------|------|
| `/clear` | 清空当前对话上下文 |
| `/compact` | 压缩历史消息，节省上下文空间 |
| `/init` | 分析项目并自动生成 CLAUDE.md |
| `/model` | 切换使用的模型 |
| `/help` | 查看所有可用命令 |

> 上下文接近限制时，先用 `/compact` 压缩，再继续工作。

### `/model opusplan` 详解

`/model opusplan` 是一个特别实用的组合模式：切换到 Claude Opus 模型，并进入 **Plan Mode（计划模式）**。

**Plan Mode 的工作方式：**

进入 Plan Mode 后，Claude 会先分析任务、制定完整计划并展示给你，**等待你确认后才开始执行**，不会立即修改任何文件。

```
/model opusplan
```

适合用在：
- 需要改动多个文件的大型重构
- 不确定影响范围、想先看方案再决定的场景
- 希望用更强的模型做复杂任务规划

**与普通 `/model` 的对比：**

| 模式 | 命令 | 特点 |
|------|------|------|
| 默认模式 | — | 直接执行，速度快 |
| 切换模型 | `/model claude-opus-4-6` | 换用 Opus，仍直接执行 |
| Opus Plan 模式 | `/model opusplan` | Opus 模型 + 先规划后执行 |

> 处理复杂任务时，先用 `/model opusplan` 让 Opus 给出完整方案，确认无误后再执行，能有效避免改错方向。

## 技巧四：逐步确认，避免误操作

Claude Code 会主动执行命令（包括删除文件、推送代码等）。新手建议：

1. **每次操作前看清楚它要做什么**，工具调用会显示具体命令
2. **涉及 Git push、删除文件等高风险操作时**，它会默认先询问确认
3. **可以随时按 `Esc` 中断正在执行的操作**

```
# 明确告诉它操作范围，减少意外
只修改 src/utils.js 这一个文件，不要动其他文件
```

## 技巧五：拆解大任务，分步执行

一次性要求太多容易出错，复杂任务拆成小步骤更可靠：

```
# 效果差：
帮我重构整个项目的认证模块，加上 JWT、刷新 Token 和权限控制

# 效果好：
第一步：先帮我分析现有认证模块的结构
（确认后）
第二步：把登录接口改成 JWT 方式
（确认后）
第三步：添加 Token 刷新逻辑
```

## 技巧六：让 Claude Code 解释它做了什么

完成一个复杂任务后，可以让它解释修改内容：

```
刚才你改了哪些文件？每个文件改了什么，为什么这样改？
```

这样既能学习，也能发现潜在问题。

---

## 技巧七：用图片作为输入

Claude Code 支持多模态输入，可以直接把截图、设计稿、错误截图拖入终端：

```
（拖入报错截图）
这个错误怎么解决？

（拖入设计图）
按这个设计图帮我写对应的 HTML/CSS
```

---

## 技巧八：配合 Git 工作流使用

Claude Code 与 Git 配合非常顺手：

```bash
# 让它帮你写 commit message
帮我根据刚才的改动写一个规范的 commit message

# 代码审查
帮我 review 这次的改动，指出潜在的 bug 和可以优化的地方

# 解决冲突
我有 merge conflict，帮我分析并解决
```

---

## 小结

| 场景 | 技巧 |
|------|------|
| 不知道命令 | 直接描述意图 |
| 重复解释项目背景 | 写好 CLAUDE.md |
| 上下文满了 | 用 `/compact` 压缩 |
| 复杂任务想先看方案 | 用 `/model opusplan` 规划后再执行 |
| 怕误操作 | 拆小步骤 + 确认后再执行 |
| 看不懂报错 | 截图拖进去 |

Claude Code 本质上是一个懂代码的协作者，越清晰地表达需求，它给出的结果越准确。多用、多实验，很快就能找到适合自己的工作方式。
